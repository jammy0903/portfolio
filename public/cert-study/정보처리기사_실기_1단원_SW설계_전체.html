1. 소프트웨어 개발 방법론

순서냐, 반복이냐, 애자일이냐

폭포수 (Waterfall)단계 순서 진행, 이전 단계 복귀 어려움
나선형 (Spiral)위험 분석 반복. 계획→분석→개발→평가 사이클
프로토타이핑견본(프로토타입) 먼저 만들어 요구사항 확인
애자일 (Agile)짧은 반복(스프린트), 변화에 유연
스크럼 (Scrum)스프린트·백로그·데일리 스크럼
XP (익스트림 프로그래밍)페어 프로그래밍, TDD, 지속적 통합
SDLC소프트웨어 개발 생명주기
CBD컴포넌트 기반 개발
클린 코드가독성·유지보수성 높은 코드
애자일 vs 폭포수: 폭포수 = 계획 중심, 문서화, 변경 어려움 / 애자일 = 작동 소프트웨어 중심, 변화 수용. 스크럼의 역할: 스프린트(2~4주 반복), 스크럼 마스터(방해 제거), 프로덕트 오너(백로그 관리).

2. 아키텍처 패턴

시스템 전체 구조 설계 방식

계층형 (Layered)상위·하위 계층 분리. OSI 모델 같은 구조
클라이언트-서버서비스 제공자와 요청자 분리
MVCModel·View·Controller 분리
파이프-필터데이터가 파이프를 통해 필터 단계 통과
이벤트 기반이벤트 발생 시 핸들러 실행
마스터-슬레이브마스터가 슬레이브에 작업 분산
MSA마이크로서비스 — 서비스별 독립 배포
브로커컴포넌트 간 중재자 역할
MVC 핵심: Model = 데이터·비즈니스 로직 / View = 화면 / Controller = 입력 처리. 시험에서 MVC 각 역할 묻는 문제 자주 출제.

3. 결합도 (Coupling) — 낮을수록 좋음

모듈 간 의존성 — 내용 → 공통 → 외부 → 제어 → 스탬프 → 자료 (강 → 약)

① 내용 결합 (Content) 한 모듈이 다른 모듈의 내부 로직·데이터를 직접 참조·수정. 최악.
② 공통 결합 (Common) 두 모듈이 전역 변수(공유 데이터) 공동 사용
③ 외부 결합 (External) 외부 환경(파일·OS·하드웨어)에 종속
④ 제어 결합 (Control) 제어 신호(플래그·스위치)를 파라미터로 전달해 처리 흐름 제어
⑤ 스탬프 결합 (Stamp) 레코드·구조체 등 자료구조 전체를 파라미터로 전달
⑥ 자료 결합 (Data) 단순 값(스칼라)만 파라미터로 전달. 최선.
암기 순서 (강 → 약): 내용·공통·외부·제어·스탬프·자료. 앞쪽일수록 나쁨. 자료 결합이 가장 이상적.

4. 응집도 (Cohesion) — 높을수록 좋음

모듈 내부 요소들의 관련성 — 기능 → 순차 → 통신 → 절차 → 시간 → 논리 → 우연 (강 → 약)

① 기능적 응집 (Functional) 단 하나의 기능만 수행. 최선.
② 순차적 응집 (Sequential) 앞 처리 출력이 다음 처리 입력. 순서 있음.
③ 통신적 응집 (Communicational) 같은 자료를 사용하는 기능들의 묶음
④ 절차적 응집 (Procedural) 관련 기능을 순서대로 수행 (자료 공유 없음)
⑤ 시간적 응집 (Temporal) 같은 시간에 실행되는 기능들의 묶음 (초기화 루틴 등)
⑥ 논리적 응집 (Logical) 유사 기능을 묶어두고 선택 플래그로 실행
⑦ 우연적 응집 (Coincidental) 아무 관련 없는 기능들이 우연히 묶임. 최악.
암기 순서 (강 → 약): 기능·순차·통신·절차·시간·논리·우연. 앞쪽이 좋고 뒤쪽이 나쁨. 결합도는 낮추고 응집도는 높여야 좋은 모듈.

5. UML 다이어그램

구조 다이어그램 / 행위 다이어그램 분류

구조 다이어그램 (정적)

  • 클래스 다이어그램 — 클래스·관계
  • 객체 다이어그램 — 인스턴스 상태
  • 컴포넌트 다이어그램 — 컴포넌트 구조
  • 배포 다이어그램 — 하드웨어 배치
  • 패키지 다이어그램 — 패키지 관계

행위 다이어그램 (동적)

  • 유스케이스 다이어그램 — 사용자·시스템 상호작용
  • 시퀀스 다이어그램 — 시간 순 메시지
  • 활동 다이어그램 — 흐름·작업 흐름도
  • 상태 다이어그램 — 상태 전이
  • 협업 다이어그램 — 객체 협력 관계
자주 나오는 관계: 연관(Association) / 의존(Dependency) / 상속(Generalization) / 실체화(Realization) / 집합(Aggregation) / 합성(Composition). 시험에서 화살표 방향과 이름을 매핑하는 문제 출제.

6. 디자인 패턴 분류 요약표

생성 / 구조 / 행위 한눈에 — 세부는 별도 페이지 참고

생성 패턴 (5개)

  • Singleton — 인스턴스 하나
  • Factory Method — 생성 서브클래스 위임
  • Abstract Factory — 객체군 생성
  • Builder — 단계적 생성
  • Prototype — 복제

구조 패턴 (7개)

  • Adapter — 인터페이스 변환
  • Bridge — 추상화·구현 분리
  • Composite — 부분·전체 동일 취급
  • Decorator — 기능 동적 추가
  • Facade — 단순 창구
  • Flyweight — 인스턴스 공유
  • Proxy — 대리 접근 제어
행위 패턴 (11개) Observer / Strategy / Command / Iterator / Visitor / Chain / Template / State / Mediator / Memento / Interpreter
시험 빈출 Observer·Proxy·Factory Method·Singleton·Visitor·Abstract Factory·Iterator·Adapter
미출제 최우선 Strategy·Decorator·Facade·Builder·Prototype
디자인 패턴 세부 내용은 1단원 디자인 패턴 상세 페이지에서 확인하세요. 결합도·응집도는 순서를 통째로 외우는 게 가장 빠릅니다.
