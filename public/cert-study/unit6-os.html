1. 스케줄링

비선점 / 선점 구분, 알고리즘 특성 암기

비선점 (한 번 주면 끝까지)

  • FCFS — 먼저 온 작업 먼저
  • SJF — 실행 시간 가장 짧은 것
  • HRN — (대기+서비스)/서비스 높은 것
  • 우선순위 — 우선순위 높은 것

선점 (CPU 뺏을 수 있음)

  • SRT — 남은 시간 가장 짧은 것
  • RR — 시간 할당량(퀀텀) 순환
  • 다단계 큐 — 큐별 정책 고정
  • 다단계 피드백 큐 — 큐 간 이동 가능
HRN 공식: (대기 시간 + 서비스 시간) ÷ 서비스 시간. 값 클수록 우선 선택. 기아(starvation) 방지 목적.

2. 교착상태 (Deadlock)

발생 조건 4가지 + 해결 방법 4가지

① 상호배제자원을 한 번에 하나만 점유
② 점유대기자원 갖고 다른 자원 기다림
③ 비선점다른 프로세스 자원 강탈 불가
④ 환형대기프로세스가 원형으로 서로 대기
예방 (Prevention)4가지 조건 중 하나 제거
회피 (Avoidance)은행가 알고리즘 — 안전 상태 유지
탐지 (Detection)교착상태 발생 후 탐지
회복 (Recovery)프로세스 종료 or 자원 선점
발생 조건 4개 = 상점비환 (상호배제·점유대기·비선점·환형대기). 은행가 알고리즘 = 회피 기법의 대표.

4. RAID (0 ~ 6 + 10)

속도냐 안정성이냐 — 레벨별 특성 한 줄씩

RAID 0 — 스트라이핑 데이터를 여러 디스크에 분산 저장. 속도 최대 / 하나 고장 시 전체 데이터 손실. 패리티 없음.
RAID 1 — 미러링 데이터를 두 디스크에 똑같이 복사. 안정성 최고 / 용량 50% 낭비. 패리티 없음.
RAID 2 — 해밍코드 비트 단위 스트라이핑 + 해밍 ECC. 비실용적 — 거의 미사용.
RAID 3 — 전용 패리티 (바이트) 바이트 단위 스트라이핑 + 별도 패리티 드라이브 1개. 패리티 드라이브 병목.
RAID 4 — 전용 패리티 (블록) 블록 단위 스트라이핑 + 별도 패리티 드라이브 1개. 역시 패리티 드라이브 병목.
RAID 5 — 분산 패리티 블록 스트라이핑 + 패리티를 모든 디스크에 분산. 가장 많이 사용. 최소 3개 디스크.
RAID 6 — 이중 분산 패리티 RAID 5 + 패리티 두 세트. 디스크 2개 동시 고장 허용. 최소 4개 디스크.
RAID 10 (1+0) RAID 1(미러링) + RAID 0(스트라이핑) 결합. 속도 + 안정성 / 비용↑
RAID 01 (0+1) RAID 0 먼저, RAID 1로 미러링. RAID 10보다 복구 취약.
한 줄 요약: RAID 0 = 속도 / RAID 1 = 미러링 / RAID 5 = 분산 패리티(실무 표준) / RAID 6 = 이중 패리티(더 안전) / RAID 10 = 속도+안전. 시험은 "분산 패리티 = RAID 5"를 가장 많이 묻는다.

5. 리눅스 / 유닉스 명령어

파일 / 권한 / 프로세스 / 네트워크 / 검색

파일 / 디렉터리

  • ls — 목록 표시 (ls -al: 숨김+상세)
  • cd — 디렉터리 이동
  • pwd — 현재 경로 출력
  • mkdir — 디렉터리 생성
  • rmdir — 빈 디렉터리 삭제
  • cp — 파일/디렉터리 복사
  • mv — 이동 / 이름 변경
  • rm — 삭제 (rm -rf: 강제 재귀)
  • cat — 파일 내용 출력
  • touch — 빈 파일 생성 / 시간 갱신
  • find — 파일 검색 (find / -name)
  • grep — 패턴 검색

권한 / 프로세스 / 네트워크

  • chmod — 권한 변경 (chmod 755)
  • chown — 소유자 변경
  • chgrp — 그룹 변경
  • ps — 프로세스 목록
  • kill — 프로세스 종료 (kill -9)
  • top — 실시간 프로세스 모니터
  • df — 디스크 사용량
  • du — 디렉터리 용량
  • free — 메모리 사용량
  • ping — 연결 확인
  • ifconfig — 네트워크 인터페이스 설정
  • netstat — 네트워크 연결 상태
chmod 숫자 방식읽기=4, 쓰기=2, 실행=1. 합산. 755 = rwxr-xr-x
파이프 (|)앞 명령 출력을 다음 명령 입력으로 전달
tar -cvf / -xvfc=압축생성 / x=압축해제 / v=목록 / f=파일명
권한 표기: rwxrwxrwx = 소유자·그룹·기타 각 3비트. chmod 777 = 모두 허용. 실행 파일은 x 비트 필요.

7. 빅데이터 / 하둡 / 저장소

빅데이터 개념 → 수집 → 저장·처리 → 분석 저장소 흐름

빅데이터 개념 — 3V / 5V + 데이터 유형
빅데이터 특성 (3V → 5V)
  • Volume (양) — 방대한 데이터 규모
  • Velocity (속도) — 빠른 생성·처리 속도
  • Variety (다양성) — 정형·비정형 다양한 형태
  • Veracity (정확성) — 데이터 신뢰·품질
  • Value (가치) — 분석을 통한 비즈니스 가치
데이터 유형 3종 — 빈출
  • 정형 데이터 — 고정 스키마. 행·열 구조. RDB 테이블, CSV.
  • 반정형 데이터 — 스키마 내포. XML, JSON, HTML.
  • 비정형 데이터 — 구조 없음. 텍스트, 이미지, 동영상, SNS.
수집 → 저장·처리 도구
수집 도구
  • Flume — 로그 데이터 수집·집계·이동
  • Sqoop — 관계형 DB ↔ 하둡 데이터 이동
  • Scrapy — 웹 크롤링 수집 프레임워크
  • Kafka — 실시간 스트리밍 메시지 큐
  • ETL — Extract(추출)·Transform(변환)·Load(적재)
저장·처리 도구
  • HDFS — 하둡 분산 파일 시스템. 대용량 분산 저장.
  • MapReduce — Map(분산처리) + Reduce(집계). 병렬 처리.
  • Spark — 인메모리 처리. MapReduce보다 빠름.
  • Hive — HDFS 위 SQL 유사 질의(HiveQL).
  • Pig — 하둡 데이터 흐름 스크립팅 언어.
분석 저장소 — DW vs DM vs Data Lake 비교 (빈출)
구분데이터 웨어하우스 (DW)데이터마트 (DM)데이터 레이크
범위전사(기업 전체) 통합부서·주제별 소규모전사 원시 데이터
데이터정제·가공된 데이터DW의 부분집합원시 데이터 그대로
스키마스키마 먼저 정의스키마 먼저 정의스키마 없음(나중에)
사용자경영진·분석가특정 부서 담당자데이터 과학자
목적BI·전략적 의사결정부서별 빠른 분석탐색적 분석·ML
OLAP — 다차원 분석 연산
Roll-up (롤업)

세부 → 요약. 일별→월별→연별 집계. 상위 레벨로 올라감.

Drill-down (드릴다운)

요약 → 세부. 연별→월별→일별. 하위 레벨로 내려감.

Slice (슬라이스)

하나의 차원을 특정 값으로 자름. 예: 2024년도만 추출.

Dice (다이스)

여러 차원에서 부분 큐브 추출. 조건 복수 적용.

Pivot (피벗)

행·열 축 회전. 데이터 뷰 전환.

빅데이터 흐름: 수집(Flume·Sqoop·Kafka) → 저장(HDFS) → 처리(MapReduce·Spark) → 분석(DW/DM/DataLake → OLAP).

8. NoSQL

NoSQL vs RDBMS 비교
구분RDBMSNoSQL
스키마고정 스키마 (테이블)유동적 스키마
확장수직 확장 (Scale-up)수평 확장 (Scale-out)

적합정형 데이터, 트랜잭션대용량·비정형, 빠른 쓰기
NoSQL 유형 4종 — 저장 형태로 암기
Key-Value Store

키/값 쌍으로 저장. 가장 단순. 캐싱·세션 저장에 적합.
Redis, DynamoDB, Memcached

Document Store

JSON·XML 형태의 문서로 저장. 스키마 자유롭게 변경 가능.
MongoDB, CouchDB

Column-Family Store

열(Column) 단위로 저장. 대용량 읽기에 유리. 분석용.
Cassandra, HBase

Graph DB

노드(개체)·엣지(관계)·속성으로 저장. 관계 분석에 특화.
Neo4j

BASE 모델: Basically Available(기본 가용) · Soft-state(일시 불일치 허용) · Eventually Consistent(최종 일관성). ACID와 반대 개념.

9. 데이터마이닝 / 데이터 용어 사전

마이닝 기법 + 빈출 데이터 용어 완전 정리 — 정의 그대로 답이 되는 파트

데이터마이닝 기법 — 대용량 데이터에서 패턴 추출
분류 (Classification)

미리 정한 범주로 데이터 분류. 의사결정나무, 나이브베이즈.

연관 (Association)

함께 발생하는 항목 간 규칙 발견. 장바구니 분석 (맥주→기저귀).

연속(순서) 패턴

시간 순서에 따른 패턴 발견. 구매 후 행동 예측.

군집화 (Clustering)

유사한 데이터끼리 그룹화. 레이블 없는 비지도 학습.

텍스트마이닝

비정형 텍스트에서 의미·패턴 추출. 감성 분석, 키워드 추출.

웹마이닝

웹 로그·구조·콘텐츠 분석. 클릭 패턴, 페이지 중요도.

핵심 데이터 용어 — 정의 통째로 암기 (빈출)
용어정의 (시험 답안)암기 포인트
다크데이터
Dark Data		 수집·저장됐지만 분석·활용되지 않고 방치된 데이터 어둠 속에 묻혀있는 데이터. 저장 비용만 발생.
메타데이터
Metadata		 데이터에 대한 데이터. 데이터의 구조·의미·관리 정보 "데이터의 데이터" 그대로 답.
데이터 레이크
Data Lake		 정형·비정형 원시 데이터를 형식 구분 없이 대규모로 저장 가공 없이 그대로. 나중에 필요할 때 꺼냄.
마이데이터
MyData		 개인이 자신의 데이터를 직접 관리·활용하는 데이터 주권 서비스 내 데이터는 내가 통제. 금융·의료 마이데이터.
오픈 데이터 누구나 자유롭게 접근·활용·재배포 가능한 공개 데이터 공공데이터포털(data.go.kr) 등.
디지털 트윈
Digital Twin		 현실 세계 객체·시스템을 가상으로 복제한 디지털 모델 공장·도시를 가상으로 만들어 시뮬레이션.
디지털 아카이빙 디지털 콘텐츠를 장기 보존하기 위해 관리·저장하는 활동 아카이브 = 기록 보관소.
데이터 리터러시 데이터를 읽고 이해하며 활용할 수 있는 능력 리터러시 = 문해력. 데이터 문해력.
스몰데이터 개인·소규모 단위에서 생성·활용되는 소량의 의미 있는 데이터 빅데이터 반대. 개인 맞춤화에 활용.
MDM
Master Data Mgmt		 기업 내 핵심 데이터(마스터 데이터)를 통합·표준화하여 관리 고객·상품 정보 일관성 유지.
AI / 머신러닝 신기술 용어
머신러닝 (ML)
