1. 비트 연산 (Bitwise Operation)

연산자 6종 · 진리표 · 실제 계산 예시 · 시험 활용 패턴

비트 연산자 6종 — 기호 + 의미 암기
연산자이름규칙의미 · 특징
&AND둘 다 1 → 1두 비트 모두 1일 때만 1. 비트 마스킹에 사용.
|OR하나라도 1 → 1하나만 1이어도 1. 비트 설정에 사용.
^XOR다르면 1, 같으면 0배타적 OR. 같은 값 두 번 XOR하면 원래 값 복원.
~NOT0→1, 1→0모든 비트 반전. 1의 보수. ~n = -(n+1).
<<Left Shift왼쪽으로 n칸 이동빈 자리는 0으로 채움. × 2ⁿ 효과.
>>Right Shift오른쪽으로 n칸 이동빈 자리는 부호비트로 채움(산술). ÷ 2ⁿ 효과.
AND · OR · XOR · NOT 진리표
ABA & BA | BA ^ B~A
000001
010111
100110
111100
AND: 둘 다 1일 때만 1  |  OR: 하나라도 1이면 1
XOR: 다를 때 1, 같을 때 0  |  NOT: 반대
실제 계산 예시 — 5 와 3으로 전부 풀기
5 = 0101  (이진수)
3 = 0011  (이진수)

5 & 3 → 0101
        0011
        ────
        0001 = 1   // AND: 둘 다 1인 자리만

5 | 3 → 0101
        0011
        ────
        0111 = 7   // OR: 하나라도 1인 자리

5 ^ 3 → 0101
        0011
        ────
        0110 = 6   // XOR: 다른 자리만 1

~5    → 00000101
        ────────
        11111010 = -6 // NOT: 전체 반전, -(5+1)
Shift 연산 예시 — 곱나누기로 이해
Left Shift (<<) — × 2ⁿ
5 << 1 :
  00000101
→ 00001010  // 왼쪽 이동, 오른쪽 0 채움
= 10  // 5 × 2¹ = 10

5 << 2 :
  000000101
→ 0000010100
= 20  // 5 × 2² = 20

// 규칙: n << k = n × 2ᵏ
Right Shift (>>) — ÷ 2ⁿ (버림)
20 >> 1 :
  00010100
→  00001010  // 오른쪽 이동
= 10  // 20 ÷ 2¹ = 10

7 >> 1 :
  00000111
→  0000011  // 1 버림
= 3   // 7 ÷ 2 = 3 (소수점 버림)

// 규칙: n >> k = n ÷ 2ᵏ (정수 버림)
시험 빈출 활용 패턴 — 코드 독해 시 즉시 인식
짝수·홀수 판별 
n & 1
0 → 짝수 (마지막 비트 0)
1 → 홀수 (마지막 비트 1)

예: 6 & 1 = 0110 & 0001
         = 0000 = 0 → 짝수
    7 & 1 = 0111 & 0001
         = 0001 = 1 → 홀수
k번째 비트 확인·설정·제거 
// k번째 비트 확인 (0부터)
n & (1 << k)
→ 0이면 꺼짐, 0아니면 켜짐

// k번째 비트 켜기(1로)
n | (1 << k)

// k번째 비트 끄기(0으로)
n & ~(1 << k)
XOR 특성 활용 
// 같은 수 XOR = 0
a ^ a = 0

// 0과 XOR = 자기 자신
a ^ 0 = a

// 두 번 XOR = 복원
(a ^ b) ^ b = a

// 스왑 (임시변수 없이)
a = a^b; b = a^b; a = a^b;
시험 출력 추적 팁: 비트 연산 문제는 10진수를 2진수로 바꿔 세로로 쓰고 한 자리씩 계산.   우선순위: ~ (높음) → << >> → & → ^ → | (낮음). 괄호로 명확히 표현된 경우가 많음.

2. Java 핵심

OOP 4대 특성 + 자주 나오는 문법

캡슐화데이터·메서드를 하나로 묶고 접근 제한
상속부모 클래스 속성·메서드 재사용
다형성같은 메서드명 → 다른 동작 (오버로딩·오버라이딩)
추상화불필요한 세부 숨기고 핵심만 표현
오버로딩같은 이름, 다른 매개변수. 같은 클래스 내
오버라이딩부모 메서드를 자식이 재정의. @Override
접근 제한자public > protected > default > private

클래스 / 인터페이스

  • abstract class: 추상 메서드 포함 가능
  • interface: 다중 구현 가능, 상수+추상메서드
  • static: 인스턴스 없이 호출
  • final: 변경 불가 (변수·메서드·클래스)

자주 나오는 API

  • System.out.println() — 출력
  • String.length() — 문자열 길이
  • String.substring(a, b) — 부분 문자열
  • Math.max(), Math.min()
  • ArrayList.add(), .get(), .size()
시험에서 자주 나오는 패턴: ① 생성자 오버로딩 ② 상속 + 오버라이딩 출력 ③ static 변수 공유 ④ this / super 사용.

4. 자료구조

선형 / 비선형 — 특성과 연산 이름 암기

선형 자료구조

  • 배열 (Array) — 연속 메모리, 인덱스 접근 O(1)
  • 스택 (Stack) — LIFO. push/pop. 재귀·DFS·괄호 검사
  • 큐 (Queue) — FIFO. enqueue/dequeue. BFS·프린터
  • 덱 (Deque) — 양쪽 삽입·삭제 가능
  • 연결 리스트 — 노드+포인터. 삽입·삭제 O(1), 탐색 O(n)

비선형 자료구조

  • 트리 (Tree) — 계층 구조. 루트·리프·차수
  • 이진 트리 — 자식 최대 2개
  • 이진 탐색 트리 (BST) — 왼쪽 < 루트 < 오른쪽
  • 힙 (Heap) — 최솟값/최댓값 빠른 접근
  • 그래프 (Graph) — 정점(V)+간선(E). 방향/무방향
  • 해시 테이블 — 키 → 해시함수 → 인덱스
전위 순회 (Preorder)루트 → 왼쪽 → 오른쪽
중위 순회 (Inorder)왼쪽 → 루트 → 오른쪽 (오름차순 정렬)
후위 순회 (Postorder)왼쪽 → 오른쪽 → 루트
스택 = LIFO (Last In First Out) / 큐 = FIFO (First In First Out). 트리 순회는 루트 위치로 기억: 전위=루트 처음 / 중위=루트 중간 / 후위=루트 마지막.

5. 정렬 알고리즘

시간복잡도 O(n²) vs O(n log n)

버블 정렬 O(n²)인접한 두 원소 비교·교환. 단순하나 느림
선택 정렬 O(n²)최솟값 찾아 맨 앞으로. 교환 횟수 최소
삽입 정렬 O(n²)앞쪽 정렬 부분에 하나씩 삽입. 거의 정렬 시 빠름
퀵 정렬 O(n log n)피벗 기준 분할. 평균 가장 빠름. 최악 O(n²)
병합 정렬 O(n log n)분할 후 병합. 안정 정렬. 추가 메모리 필요
힙 정렬 O(n log n)힙 자료구조 활용. 추가 메모리 불필요
카운팅 정렬 O(n+k)원소 범위 작을 때 O(n) 가능
기수 정렬 O(nk)자릿수별 정렬. 비교 없음
안정 정렬같은 값 → 원래 순서 유지. 병합·버블·삽입
시간복잡도 암기: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ). 공간복잡도: 제자리 정렬(in-place) = 추가 공간 O(1).

6. 탐색 알고리즘

순차 / 이진 / 그래프 탐색

순차 탐색 O(n)처음부터 끝까지 순서대로 비교
이진 탐색 O(log n)정렬된 배열 필수. 중간값 비교 후 절반 제거
DFS (깊이 우선)스택 또는 재귀. 끝까지 파고들다 백트래킹
BFS (너비 우선)큐 사용. 최단 경로 탐색에 유리
다익스트라가중치 그래프 최단 경로 (음수 없음)
벨만-포드음수 가중치 허용 최단 경로
해시 탐색 O(1)해시 함수로 직접 접근. 평균 O(1)
BFS = 큐 / DFS = 스택(재귀). 최단 경로 = BFS(비가중) or 다익스트라(가중). 이진 탐색은 반드시 정렬 전제.
