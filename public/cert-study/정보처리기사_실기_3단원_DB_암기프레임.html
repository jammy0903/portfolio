2. 키(Key) 종류

유일성 / 최소성 / 역할로 구분

슈퍼키 (Super Key)유일성 만족. 최소성 불필요. 가장 상위 개념
후보키 (Candidate Key)유일성 + 최소성. 기본키 후보군
기본키 (Primary Key)후보키 중 선택. NULL 불가. 중복 불가
대체키 (Alternate Key)후보키 중 기본키가 아닌 나머지
외래키 (Foreign Key)다른 테이블의 기본키 참조. 참조 무결성
복합키 (Composite Key)두 개 이상 속성 조합으로 식별
포함 관계: 슈퍼키 ⊃ 후보키 ⊃ 기본키. 대체키 = 후보키 - 기본키. 기본키는 NULL 불가 / 외래키는 NULL 가능(참조 무결성 조건 충족 시).

3. 정규화 (1NF ~ BCNF)

이상(Anomaly) 제거가 목적 — 단계별 조건 암기

정규화 단계

  1. 1NF — 모든 속성이 원자값 (반복 그룹 제거)
  2. 2NF — 부분 함수 종속 제거 (완전 함수 종속)
  3. 3NF — 이행 함수 종속 제거
  4. BCNF — 결정자 모두가 후보키
  5. 4NF — 다치 종속 제거
  6. 5NF — 조인 종속 제거
시험은 주로 1~BCNF 범위.

이상(Anomaly) 종류

  • 삽입 이상 — 불필요한 데이터 함께 삽입
  • 삭제 이상 — 삭제 시 필요한 데이터도 사라짐
  • 갱신 이상 — 일부 수정 시 불일치 발생
역정규화: 성능 향상을 위해 의도적으로 정규화를 되돌리는 것.

함수 종속 개념

  • 함수 종속 (FD) — X → Y: X가 Y를 결정
  • 완전 함수 종속 — 기본키 전체에 종속
  • 부분 함수 종속 — 기본키 일부에 종속 (2NF 위반)
  • 이행 함수 종속 — A→B, B→C면 A→C (3NF 위반)

6. SQL — DDL / DML / DCL / TCL

명령어 분류 먼저, 각 문법 뒤에 붙이기

DDL (데이터 정의어)

  • CREATE TABLE 테이블명 (컬럼명 타입 [제약], ...)
  • ALTER TABLE 테이블명 ADD/MODIFY/DROP 컬럼
  • DROP TABLE 테이블명 — 테이블 삭제
  • TRUNCATE TABLE — 데이터 전체 삭제 (구조 유지)
  • CREATE INDEX ON 테이블명(컬럼)
  • CREATE VIEW 뷰명 AS SELECT...

DML / DCL / TCL

  • INSERT INTO 테이블 VALUES (...)
  • UPDATE 테이블 SET 컬럼=값 WHERE 조건
  • DELETE FROM 테이블 WHERE 조건
  • GRANT 권한 ON 테이블 TO 사용자
  • REVOKE 권한 ON 테이블 FROM 사용자
  • COMMIT — 트랜잭션 확정
  • ROLLBACK — 트랜잭션 취소
  • SAVEPOINT 이름 — 중간 저장점
무결성 제약NOT NULL / UNIQUE / PRIMARY KEY / FOREIGN KEY / CHECK / DEFAULT
CASCADE부모 삭제 시 자식도 함께 삭제
뷰 (VIEW)가상 테이블. 실제 데이터 없음. 보안·편의용
DDL vs DML: DDL = 구조(테이블) 정의 → 자동 COMMIT / DML = 데이터 조작 → 명시적 COMMIT 필요. DROP은 테이블 삭제, TRUNCATE는 데이터만 삭제(구조 유지), DELETE는 조건부 삭제.

7. SQL SELECT 완전 정리

실기에서 가장 많이 출제 — 실행 순서와 JOIN 종류 필수

SELECT 기본 구조 & 실행 순서

  • SELECT 컬럼 (⑤ 출력)
  • FROM 테이블 (① 테이블 읽기)
  • WHERE 조건 (② 행 필터)
  • GROUP BY 컬럼 (③ 그룹화)
  • HAVING 집계조건 (④ 그룹 필터)
  • ORDER BY 컬럼 ASC/DESC (⑥ 정렬)
WHERE는 그룹화 전 / HAVING은 그룹화 후 필터. GROUP BY 없으면 HAVING 못 씀.

집계 함수 / 기타

  • COUNT(*) — 행 수 (NULL 포함)
  • COUNT(컬럼) — NULL 제외 행 수
  • SUM — 합계
  • AVG — 평균
  • MAX / MIN — 최댓값 / 최솟값
  • DISTINCT — 중복 제거
  • AS — 별칭 지정
  • LIKE '%값%' — 패턴 검색
  • BETWEEN A AND B — 범위 검색
  • IN (값1, 값2) — 목록 중 하나
  • IS NULL / IS NOT NULL — NULL 검사
INNER JOIN두 테이블 교집합 (일치하는 행만)
LEFT (OUTER) JOIN왼쪽 테이블 전부 + 오른쪽 일치
RIGHT (OUTER) JOIN오른쪽 전부 + 왼쪽 일치
FULL OUTER JOIN양쪽 전부 (불일치는 NULL)
CROSS JOIN카티션 프로덕트 (모든 조합)
SELF JOIN같은 테이블끼리 조인
서브쿼리 (Subquery)SELECT 안에 SELECT. WHERE/FROM/SELECT 절에 사용 가능
UNION / UNION ALL결과 합치기. UNION은 중복 제거, UNION ALL은 중복 포함
자주 나오는 패턴: ① GROUP BY + HAVING으로 조건 집계 ② LEFT JOIN으로 없는 데이터 포함 ③ 서브쿼리로 중첩 조건 ④ COUNT(DISTINCT 컬럼)으로 중복 없는 카운트.

8. 트랜잭션

ACID + 회복 + 병행제어

AAtomicity원자성 — 전부 하거나 전혀 안 하거나
CConsistency일관성 — 전후 상태가 일관
IIsolation격리성 — 서로 간섭 금지
DDurability영속성 — 완료 결과는 영구 저장

회복 기법

  • REDO — 커밋된 것 다시 반영
  • UNDO — 미완료 것 취소
  • 즉각 갱신 — 변경 즉시 DB 반영
  • 지연 갱신 — 커밋 시점에 반영
  • 검사점 (Checkpoint) — 주기적 저장점

병행 제어

  • Locking — 자원 잠그기
  • 2단계 잠금 (2PL) — 확장 + 수축 단계
  • 낙관적 검증 — 끝에서 충돌 검사
  • 타임스탬프 순서 — 시간 기준 직렬화
  • MVCC — 버전 여러 개 유지
DB는 SQL SELECT가 실기에서 가장 자주 나옵니다. GROUP BY/HAVING 차이, LEFT JOIN, 서브쿼리는 반드시 직접 써보면서 익히세요.
