1. OSI 7계층 / 프로토콜 기본

계층 문제와 구문·의미·타이밍

7 응용 (Application)
6 표현 (Presentation)
5 세션 (Session)
4 전송 (Transport)
3 네트워크 (Network)
2 데이터링크 (Data Link)
1 물리 (Physical)
  • 물리 = 비트 신호 전송
  • 데이터링크 = MAC 주소, 오류 제어 (프레임)
  • 네트워크 = IP 주소, 라우팅 (패킷)
  • 전송 = 종단 간 신뢰성, TCP/UDP (세그먼트)
  • 세션 = 연결 설정·유지·동기화
  • 표현 = 형식 변환·암호화·압축
  • 응용 = 사용자 서비스 (HTTP, FTP 등)
구문 (Syntax)형식·구조
의미 (Semantic)각 필드의 뜻
타이밍 (Timing)순서·속도·시점

2. TCP/IP 4계층 (인터넷 모델)

OSI와 대응 관계 필수

응용 계층OSI 5·6·7층 대응. HTTP, FTP, DNS, SMTP
전송 계층OSI 4층 대응. TCP, UDP
인터넷 계층OSI 3층 대응. IP, ICMP, ARP, RARP
네트워크 접속 계층OSI 1·2층 대응. Ethernet, Wi-Fi

TCP (연결형)

  • 신뢰성 보장, 순서 보장
  • 3-way handshake (연결)
  • 4-way handshake (해제)
  • 흐름제어, 혼잡제어

UDP (비연결형)

  • 신뢰성 없음, 빠름
  • 실시간 스트리밍, DNS, DHCP
  • 헤더 단순 (8바이트)
  • 멀티캐스트 지원
3-way handshake: SYN → SYN-ACK → ACK. 4-way handshake(해제): FIN → ACK → FIN → ACK.

8. 서브넷 계산 공식

블록크기 = 256 − 마스크값 (이진법 없이 공식 하나로)

① 공식 3줄

  1. 블록크기 = 256 − 마스크값
  2. 네트워크주소 마지막옥텟
    = (IP마지막옥텟 ÷ 블록크기) 버림 × 블록크기
  3. 브로드캐스트 = 네트워크주소 + 블록크기 − 1
  4. 사용 가능 호스트수 = 블록크기 − 2
핵심: 255 옥텟은 건드리지 말고, 255 아닌 옥텟만 계산!

② 자주 나오는 마스크

마스크값블록크기CIDR호스트수
128128/25126

③ 계산 예시

예1) 223.13.234.132 / 255.255.255.192
블록크기 = 256 − 192 = 64
132 ÷ 64 = 2.06 → 버림 → 2 × 64 = 128
→ 네트워크: 223.13.234.128
→ 브로드캐스트: 223.13.234.191
⚠️ 234는 3번째 옥텟! 건드리지 마
예2) 10.0.0.100 / 255.255.255.240
블록크기 = 256 − 240 = 16
100 ÷ 16 = 6.25 → 버림 → 6 × 16 = 96
→ 네트워크: 10.0.0.96
→ 브로드캐스트: 10.0.0.111
