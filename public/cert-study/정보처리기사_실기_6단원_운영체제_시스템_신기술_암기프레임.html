1. 스케줄링

비선점 / 선점 구분, 알고리즘 특성 암기

비선점 (한 번 주면 끝까지)

  • FCFS — 먼저 온 작업 먼저
  • SJF — 실행 시간 가장 짧은 것
  • HRN — (대기+서비스)/서비스 높은 것
  • 우선순위 — 우선순위 높은 것

선점 (CPU 뺏을 수 있음)

  • SRT — 남은 시간 가장 짧은 것
  • RR — 시간 할당량(퀀텀) 순환
  • 다단계 큐 — 큐별 정책 고정
  • 다단계 피드백 큐 — 큐 간 이동 가능
HRN 공식: (대기 시간 + 서비스 시간) ÷ 서비스 시간. 값 클수록 우선 선택. 기아(starvation) 방지 목적.

2. 교착상태 (Deadlock)

발생 조건 4가지 + 해결 방법 4가지

① 상호배제자원을 한 번에 하나만 점유
② 점유대기자원 갖고 다른 자원 기다림
③ 비선점다른 프로세스 자원 강탈 불가
④ 환형대기프로세스가 원형으로 서로 대기
예방 (Prevention)4가지 조건 중 하나 제거
회피 (Avoidance)은행가 알고리즘 — 안전 상태 유지
탐지 (Detection)교착상태 발생 후 탐지
회복 (Recovery)프로세스 종료 or 자원 선점
발생 조건 4개 = 상점비환 (상호배제·점유대기·비선점·환형대기). 은행가 알고리즘 = 회피 기법의 대표.

3. 프로세스 상태 / 페이지 교체

상태 전이, 교체 알고리즘, 리눅스 기초

준비 (Ready)CPU 할당 대기
실행 (Running)CPU 수행 중
대기 (Waiting)I/O 등 이벤트 대기
FIFO먼저 들어온 페이지 교체
LRU가장 오래 사용 안 한 페이지
LFU사용 빈도 가장 낮은 페이지
OPT가장 늦게 참조될 페이지 (최적)
NUR최근 미사용 / 변경 비트 기준
2차 기회FIFO + 참조 비트 재확인
스래싱 (Thrashing)페이지 부재 잦아 CPU 효율 급감
워킹셋 (Working Set)자주 사용하는 페이지 집합 유지
IPC(프로세스 간 통신) 방법: 파이프, 메시지 큐, 공유 메모리, 소켓, 세마포어.

4. RAID (0 ~ 6 + 10)

속도냐 안정성이냐 — 레벨별 특성 한 줄씩

RAID 0 — 스트라이핑 데이터를 여러 디스크에 분산 저장. 속도 최대 / 하나 고장 시 전체 데이터 손실. 패리티 없음.
RAID 1 — 미러링 데이터를 두 디스크에 똑같이 복사. 안정성 최고 / 용량 50% 낭비. 패리티 없음.
RAID 2 — 해밍코드 비트 단위 스트라이핑 + 해밍 ECC. 비실용적 — 거의 미사용.
RAID 3 — 전용 패리티 (바이트) 바이트 단위 스트라이핑 + 별도 패리티 드라이브 1개. 패리티 드라이브 병목.
RAID 4 — 전용 패리티 (블록) 블록 단위 스트라이핑 + 별도 패리티 드라이브 1개. 역시 패리티 드라이브 병목.
RAID 5 — 분산 패리티 블록 스트라이핑 + 패리티를 모든 디스크에 분산. 가장 많이 사용. 최소 3개 디스크.
RAID 6 — 이중 분산 패리티 RAID 5 + 패리티 두 세트. 디스크 2개 동시 고장 허용. 최소 4개 디스크.
RAID 10 (1+0) RAID 1(미러링) + RAID 0(스트라이핑) 결합. 속도 + 안정성 / 비용↑
RAID 01 (0+1) RAID 0 먼저, RAID 1로 미러링. RAID 10보다 복구 취약.
한 줄 요약: RAID 0 = 속도 / RAID 1 = 미러링 / RAID 5 = 분산 패리티(실무 표준) / RAID 6 = 이중 패리티(더 안전) / RAID 10 = 속도+안전. 시험은 "분산 패리티 = RAID 5"를 가장 많이 묻는다.

5. 리눅스 / 유닉스 명령어

파일 / 권한 / 프로세스 / 네트워크 / 검색

파일 / 디렉터리

  • ls — 목록 표시 (ls -al: 숨김+상세)
  • cd — 디렉터리 이동
  • pwd — 현재 경로 출력
  • mkdir — 디렉터리 생성
  • rmdir — 빈 디렉터리 삭제
  • cp — 파일/디렉터리 복사
  • mv — 이동 / 이름 변경
  • rm — 삭제 (rm -rf: 강제 재귀)
  • cat — 파일 내용 출력
  • touch — 빈 파일 생성 / 시간 갱신
  • find — 파일 검색 (find / -name)
  • grep — 패턴 검색

권한 / 프로세스 / 네트워크

  • chmod — 권한 변경 (chmod 755)
  • chown — 소유자 변경
  • chgrp — 그룹 변경
  • ps — 프로세스 목록
  • kill — 프로세스 종료 (kill -9)
  • top — 실시간 프로세스 모니터
  • df — 디스크 사용량
  • du — 디렉터리 용량
  • free — 메모리 사용량
  • ping — 연결 확인
  • ifconfig — 네트워크 인터페이스 설정
  • netstat — 네트워크 연결 상태
chmod 숫자 방식읽기=4, 쓰기=2, 실행=1. 합산. 755 = rwxr-xr-x
파이프 (|)앞 명령 출력을 다음 명령 입력으로 전달
tar -cvf / -xvfc=압축생성 / x=압축해제 / v=목록 / f=파일명
권한 표기: rwxrwxrwx = 소유자·그룹·기타 각 3비트. chmod 777 = 모두 허용. 실행 파일은 x 비트 필요.

6. 시스템 / 신기술 용어

RTO/RPO, 클라우드, 컨테이너, 블록체인

RTO (복구 목표 시간)장애 후 서비스 복구까지 허용 최대 시간
RPO (복구 목표 시점)복구 시 허용 가능한 데이터 손실 기준 시점
IaaS인프라 제공 (VM, 스토리지)
PaaS플랫폼 제공 (개발 환경)
SaaS소프트웨어 제공 (Gmail 등)
Docker컨테이너 기반 경량 가상화
Kubernetes컨테이너 오케스트레이션 관리
MSA마이크로서비스 아키텍처
블록체인블록 단위 분산 저장, 위변조 어려움
IoT사물 인터넷 — 기기 간 연결·통신
엣지 컴퓨팅데이터를 중앙 서버 대신 단말 근처에서 처리
컨테이너 vs VM: 컨테이너는 OS 커널 공유(가볍고 빠름) / VM은 하이퍼바이저 위에 별도 OS(무겁지만 격리 강함).

8. NoSQL

비관계형 DB — 저장 형태로 유형 외우기

Key-Value Store키/값 쌍 저장. Redis, DynamoDB
Document StoreJSON/XML 문서. MongoDB, CouchDB
Column-Family Store열 단위 저장. Cassandra, HBase
Graph DB노드·엣지 관계. Neo4j
NoSQL 특징: 고정 스키마 없음 / 수평 확장 용이 / 대용량 처리. 외우는 순서: 키값 / 문서 / 열 / 그래프. 저장 형태 설명 → 유형 이름 바로 연결.

9. 데이터마이닝 / 데이터 용어

패턴 추출 기법 + 데이터 관련 개념

분류 규칙미리 정한 범주로 분류
연관 규칙함께 발생하는 패턴 (장바구니 분석)
연속 데이터 규칙시간 순서 패턴
군집화 규칙유사한 데이터 그룹화
텍스트마이닝비정형 텍스트에서 패턴 추출
웹마이닝웹 로그·구조·콘텐츠 분석
메타데이터데이터에 대한 데이터 (구조·의미 설명)
다크데이터수집됐지만 분석·활용 안 된 데이터
디지털아카이빙디지털 콘텐츠 장기 보존
마이데이터본인 데이터 주권 보장 서비스
데이터 용어는 뜻이 거의 그대로 답입니다. "데이터에 대한 데이터" = 메타데이터, "수집만 되고 방치" = 다크데이터로 짧은 정의를 통째로 외우세요.
