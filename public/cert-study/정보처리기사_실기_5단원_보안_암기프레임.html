1. 암호 분류

대칭키냐, 비대칭키냐, 해시냐부터 구분

대칭키같은 키로 암호화 / 복호화
비대칭키공개키 / 개인키 분리
해시임의 길이 → 고정 길이 값

대칭키 쪽

  • AES = 고속 대표 블록 암호
  • DES = 56비트 고전 블록 암호
  • 3DES = DES를 세 번
  • ARIA = 국내 표준
  • SEED = 국내 개발 128비트
  • IDEA / SKIPJACK / RC4

비대칭 / 해시 쪽

  • RSA = 대표 공개키 암호
  • ECC = 타원곡선 기반
  • ElGamal = 이산로그 기반
  • Diffie-Hellman = 키 교환
  • MD5 = 128비트
  • SHA-1 = 160비트 / SHA-256 = 256비트
실기에서는 `AES / RSA / SHA-256 / Diffie-Hellman`처럼 “분류 + 대표 이름”을 같이 묻는 경우가 많다.

2. 인증 / 접근통제

누구냐, 뭐 할 수 있냐, 누가 정하냐

Authentication사용자가 누구인지 확인
Authorization허용 범위 결정
Accounting행위와 사용 내역 기록
SSO / OTP한 번 로그인 / 1회용 비밀번호
OAuth비밀번호 공유 없이 권한 위임
DAC객체 소유자가 권한 설정
MAC보안 등급 기준 강제 통제
RBAC역할 기준 권한 부여
외우는 순서: Authentication → Authorization → Accounting. 그리고 접근통제는 DAC / MAC / RBAC 3개로 묶는다.
