1. 암호화 분류

대칭키 / 비대칭키 / 해시 — 분류 먼저, 이름 나중

대칭키같은 키로 암·복호화. 빠름
비대칭키공개키(암호화) / 개인키(복호화)
해시임의 길이 → 고정 길이. 단방향

대칭키 알고리즘

  • AES — 128/192/256비트, 현 표준
  • DES — 56비트, 취약 (레거시)
  • 3DES — DES를 3번 (168비트)
  • ARIA — 국내 표준 (128/192/256)
  • SEED — 국내 개발, 128비트 블록
  • RC4 — 스트림 암호, WEP에 사용
  • Blowfish — 32~448비트 가변 키
  • IDEA — 128비트 키, 64비트 블록

비대칭키 / 해시

  • RSA — 소인수분해 기반 대표 공개키
  • ECC — 타원곡선, 짧은 키로 강도↑
  • ElGamal — 이산로그 기반
  • Diffie-Hellman — 키 교환 전용
  • MD5 — 128비트 해시 (취약)
  • SHA-1 — 160비트 (취약)
  • SHA-256 — 256비트, 현재 표준
  • SHA-512 — 512비트
  • HAS-160 — 국내 160비트 해시
구분 핵심: 키 하나 = 대칭 / 키 두 개(공개+개인) = 비대칭 / 복호화 불가 = 해시. Diffie-Hellman은 "키 교환"이지 암호화 자체가 아님.

2. 인증 / 접근통제

누구냐 → 뭐 할 수 있냐 → 누가 권한 정하냐

Authentication신원 확인 (로그인)
Authorization접근 범위 결정
Accounting사용 내역 기록
SSO한 번 로그인 → 여러 시스템 접근
OTP1회용 비밀번호 (HOTP/TOTP)
OAuth비밀번호 공유 없이 권한 위임
Kerberos티켓 기반 중앙 인증
DAC객체 소유자가 권한 설정 (임의)
MAC보안 등급 기준 강제 통제 (강제)
RBAC역할(Role) 기준 권한 부여
접근통제 구분: DAC = 소유자가 결정 / MAC = 관리자가 등급으로 강제 / RBAC = 역할에 권한 묶음. 시험은 "DAC에서 권한을 설정하는 주체"를 자주 묻는다 → 소유자.

3. CIA / 보안 속성 / 관리 도구

기밀성 / 무결성 / 가용성 + 제도 / 프로토콜

기밀성인가된 사용자만 접근
무결성허가 없이 변경 불가
가용성필요할 때 언제든 사용 가능
부인방지 (Non-repudiation)행위 사실을 나중에 부인 못 하게 함
인증 (Authentication)실체 확인
ISMS정보보호 관리체계 인증
ISMS-PISMS + 개인정보보호
SIEM보안 이벤트 수집·분석·관제
IPSecIP 계층 인증·무결성·암호화
SSL/TLS전송 계층 암호화 (HTTPS)
PKI공개키 기반 구조 (인증서)
PKI 구성: CA(인증기관) → 인증서 발급 → 공개키 배포. 전자서명 = 개인키로 서명, 공개키로 검증.

4. 방화벽 / IDS / IPS / 기타 보안 장비

차단이냐 탐지냐, 사전이냐 사후냐

방화벽 (Firewall)허가된 트래픽만 통과 / 미허가 차단
IDS침입 탐지 — 로그·알림 (사후)
IPS침입 방지 — 실시간 차단 (사전)

방화벽 유형

  • 패킷 필터링 — IP/포트 기준 차단
  • 상태 기반 검사 — 연결 상태 추적
  • 프록시 방화벽 — 내용 검사 후 전달
  • 차세대 방화벽(NGFW) — 앱 레벨 차단

기타 보안 장비

  • UTM — 방화벽+IDS+VPN 통합
  • WAF — 웹 애플리케이션 방화벽
  • DLP — 데이터 유출 방지
  • ESM — 통합 보안 관제
  • NAC — 네트워크 접근 통제
  • VPN — 암호화 터널
IDS vs IPS: IDS는 탐지만(알람), IPS는 탐지+차단. IDS가 먼저, IPS는 IDS에 차단 기능 추가. DMZ = 내부망/외부망 사이 완충 구간.

5. 웹 / 네트워크 공격

입력·세션·스크립트·위장·재전송

SQL Injection입력값에 악성 SQL 삽입 → DB 조작
XSS악성 스크립트 삽입 → 클라이언트 실행
CSRF로그인 상태 사용자 권한 악용 요청
Session Hijacking유효 세션 ID 탈취
Replay Attack정상 인증 메시지를 캡처해 재전송
Evil Twin정상 AP와 동일 SSID의 가짜 무선 AP
ARP 스푸핑거짓 ARP 응답 → MAC 테이블 오염
IP 스푸핑출발지 IP를 위조해 전송
DNS 스푸핑거짓 DNS 응답 → 가짜 사이트 유도
Sniffing패킷 몰래 가로채기
Man-in-the-Middle통신 사이에 끼어 도청·변조
타이포스쿼팅오타 도메인 선점
XSS vs CSRF: XSS는 사이트가 피해자에게 악성 스크립트 실행 / CSRF는 피해자의 권한으로 사이트에 악성 요청. 주체 방향이 반대다.

6. 악성코드 전체

감염 방식 / 동작 방식으로 분류 — 이름 + 한 줄 정의 세트로 외우기

바이러스 (Virus)다른 실행 파일에 기생, 숙주 없이 못 퍼짐
웜 (Worm)스스로 복제·전파. 숙주 파일 불필요
트로이목마 (Trojan)정상 프로그램인 척 숨은 악성 기능
랜섬웨어 (Ransomware)파일 암호화 후 복호화 대가로 금전 요구
루트킷 (Rootkit)흔적 숨기고 관리자 권한 유지
스파이웨어 (Spyware)사용자 정보를 몰래 수집·전송
애드웨어 (Adware)광고 강제 노출 프로그램
키로거 (Keylogger)키보드 입력(비밀번호 등) 기록
백도어 (Backdoor)정상 인증 우회하는 비밀 접근 경로
드로퍼 (Dropper)악성코드를 시스템에 설치하는 도구
봇 / 좀비 (Bot)원격 제어 가능한 피감염 PC
스케어웨어 (Scareware)공포심·불안 유발로 설치·결제 유도
핵심 구분: 기생형 = 바이러스 / 자기복제 = 웜 / 위장형 = 트로이목마 / 암호화 금전 = 랜섬웨어 / 권한 은닉 = 루트킷 / 정보 수집 = 스파이웨어 / 입력 기록 = 키로거 / 비밀 통로 = 백도어 / 설치 도구 = 드로퍼

7. DoS / DDoS 공격 유형

패킷 조작 / 연결 남발 / 분산 / 표적형

LAND Attack출발지 IP = 목적지 IP로 패킷 전송
Smurf Attack브로드캐스트에 ICMP 전송 → 응답 폭주
SYN FloodingTCP SYN만 계속 보내 연결 큐 고갈
Ping of Death최대 크기 초과 ICMP 패킷 전송
Teardrop패킷 오프셋 조작 → 재조합 오류 유발
UDP FloodingUDP 패킷 대량 전송
DDoS좀비 PC 다수로 분산 서비스 거부
APT장기간 은밀한 표적 공격 (Advanced Persistent Threat)
Watering Hole표적이 자주 방문하는 사이트 감염
DoS = 단일 공격자 / DDoS = 봇넷(좀비 PC 다수) 활용. LAND = 같은 IP, Smurf = 브로드캐스트 증폭, SYN Flooding = 3-way handshake 미완성.

8. 무선 보안 / 기타

WEP → WPA → WPA2 → WPA3 진화 흐름

WEPRC4 기반. 취약 — 사용 금지
WPATKIP 기반. WEP 개선 (동적 키)
WPA2AES/CCMP 기반. 현재 표준
WPA3SAE 기반. 최신 (개인키 노출 방어)
TKIPWPA에서 사용, 동적 키 변경
CCMPWPA2에서 사용, AES 기반
802.1X포트 기반 네트워크 접근 통제
TrustZoneARM 보안 영역 분리 (TEE)
템퍼프루핑하드웨어 변조 방지
다크 데이터수집됐지만 분석·활용 안 된 데이터
WEP(RC4·취약) → WPA(TKIP) → WPA2(AES·현 표준) → WPA3(SAE·최신). 시험에서 TKIP = WPA, CCMP = WPA2 짝으로 묻는다.
보안은 용어 수가 많아 한 번에 외우기 어렵습니다. 암호(대칭·비대칭·해시) → 인증·접근통제 → CIA → 방어 장비 → 공격 유형 → 악성코드 순으로 분류 먼저, 세부 이름 나중에 붙이세요.
